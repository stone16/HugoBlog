## Vision

### 10x Check
Transform stometa.top from an isolated publishing endpoint into a **personal brand operating system**. Every post auto-generates social cards, readers can discover all profiles via a Links Hub, SEO infrastructure compounds organic growth, and polished UX (progress bar, back-to-top, copy link, series nav) signals professional credibility. The blog becomes the center of a connected web: Blog <-> Twitter (@stometaverse) <-> GitHub <-> LinkedIn <-> ZentaStone.com.

### Platonic Ideal
The site feels like a media brand, not a personal blog. Landing on it, a reader immediately senses: professional credibility (founder story, company link), content depth (series navigation, archives), community (working comments), and a growth engine (every visit has a clear path to deeper connection). Emotional response: "This person is serious. I want to follow their work."

### Component Specifications

#### Progress Bar
- Height: **2px**, no track (transparent background)
- Fill: `var(--accent-grad)` (purple → cyan gradient)
- Position: `fixed`, `top: 0`, `z-index: 101` (above nav at z-100)
- Width: tracks scroll percentage via JS
- Transition: `width 0.1s linear` (must feel responsive, no spring easing here)
- Hide when: page content height < viewport height (no scrollable content)
- Semantics: `role="progressbar"`, `aria-label="Reading progress"`, `aria-valuenow/min/max`

#### Back-to-Top Button
- Size: **40px × 40px** desktop, **44px × 44px** mobile (touch target)
- Shape: rounded square (`border-radius: var(--radius-md)` = 10px), NOT circle
- Background: `var(--bg-raised)`, border: `1px solid var(--border-default)`
- Icon: chevron-up, 18px, `var(--text-muted)`
- Position: `fixed`, `bottom: 2rem`, `right: 2rem` (1.5rem / 1rem on smaller screens)
- Shadow: `var(--shadow-sm)`
- Show threshold: scroll > 300px
- Show/hide: `opacity 0→1`, `pointer-events none→auto`, `transition: 0.3s var(--ease)`
- Hover: border → `--border-strong`, icon → `--text-primary`, bg → `--bg-overlay`, `translateY(-2px)`
- Active: `scale(0.95)` spring-back
- Click: `window.scrollTo({ top: 0, behavior: 'smooth' })`
- Keyboard: focusable via Tab, Enter triggers scroll
- ARIA: `aria-label="Back to top"`

#### Copy Link Button
- Position: inline with post meta, after reading time separator
- Icon: link icon, 18px, `var(--text-muted)`
- Hover: icon → `var(--text-primary)`, cursor: pointer
- Click success: icon swaps to checkmark, `var(--accent-2)` color, tooltip "Copied!" appears above
- Tooltip: `var(--bg-overlay)`, `1px solid var(--border-default)`, `border-radius: var(--radius-sm)`, font `0.75rem`, fades out after 2 seconds
- Fallback: if Clipboard API unavailable, show "Link: [url]" as selectable text
- Keyboard: focusable, Enter triggers copy
- ARIA: `aria-label="Copy post link"`, `role="button"`, tooltip uses `aria-live="polite"`

#### Series Navigation
- Position: after post content, before share buttons
- Container: `var(--bg-card)`, `1px solid var(--border-subtle)`, `border-radius: var(--radius-lg)`, padding `1.25rem`
- Label: "Series: [series name]" in eyebrow style (uppercase, letter-spaced, 0.78rem, `--text-muted`)
- Layout: two-column (prev | next) on desktop, stacked on mobile
- Prev/Next: title text in `var(--font-display)`, 0.95rem, `--text-secondary`, max 2 lines with ellipsis
- Arrows: ← → in `--text-muted`, hover → `--accent-1`
- Hover: each nav item lifts `translateY(-2px)`, border brightens
- When only 1 post in series: render badge only, no prev/next links
- When post is not in a series: **hide entirely** (partial doesn't render)
- ARIA: each link has `aria-label="Previous in series: [title]"` / `aria-label="Next in series: [title]"`

#### Links Hub Cards
- Each card: `var(--bg-card)`, `1px solid var(--border-subtle)`, `border-radius: var(--radius-lg)`, padding `1rem 1.25rem`
- Layout: icon (20px, `--text-muted`) + label (`--font-body`, 1rem, `--text-primary`) + optional subtitle (0.85rem, `--text-secondary`)
- Hover: lift `translateY(-3px)`, `box-shadow: var(--shadow-lg), var(--shadow-glow)`, gradient border (same as post-entry hover)
- Max-width: 480px centered, gap: 0.75rem between cards
- Avatar above links: gradient "S" in 64px rounded square (radius 20%), centered
- Name: `--font-display`, 1.5rem, `--text-primary`
- Title: 0.9rem, `--text-secondary`
- All links open in new tab (`target="_blank"`, `rel="noopener noreferrer"`)

#### Favicon
- Shape: rounded square (radius ~20%)
- Background: `var(--accent-grad)` (purple → cyan, 135deg)
- Letter: "S", white (#ffffff), bold, centered
- Formats: SVG (primary), PNG 32x32 (favicon), PNG 180x180 (apple-touch-icon)
- Link tags: `<link rel="icon" type="image/svg+xml" href="/favicon.svg">` + PNG fallbacks

### Accessibility Specs

- All interactive elements: focusable via Tab, activatable via Enter
- Back-to-top: `aria-label="Back to top"`
- Copy link: `aria-label="Copy post link"`, `role="button"`, tooltip `aria-live="polite"`
- Progress bar: `role="progressbar"`, `aria-valuenow`, `aria-valuemin=0`, `aria-valuemax=100`
- Series nav links: `aria-label="Previous/Next in series: [title]"`
- Links page cards: standard `<a>` elements with descriptive text
- Touch targets: minimum 44px on mobile for all buttons
- Footer social icons: `aria-label` per icon ("Twitter", "GitHub", etc.)
